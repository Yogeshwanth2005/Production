SOGFusion Production Modules: QC, Safety, and Inventory

1. The Big Picture

Your project is called SOGFusion, a Production Dashboard for a gas filling plant. Think of it like a restaurant:

  • The Menu (React Frontend) — What you see on screen and interact with.
  • The Kitchen (Python FastAPI Backend) — Processes your requests.
  • The Recipe Book (MySQL Database) — Permanently stores all the data.

2. Quality Check Module

After cylinders are filled, an inspector checks each one. This module allows them to review pending cylinders and submit safety metrics.

"After cylinders are filled, the Quality Check module lets the inspector verify each cylinder. They check gas purity percentage, pressure, leakage, and valve condition. Each result is saved to the MySQL database through our FastAPI backend. Once checked, cylinders move to 'QC Passed' or 'QC Failed'."

How It Works:

  • Filters Data: Shows only cylinders with itemStatus = 'Produced' (waiting for QC).
  • Records Metrics: Gas Purity (%), Pressure (Pass/Fail), Leak Test, Valve Condition, and overall QC Decision.
  • Database Action: clicking "Submit QC" sends a PUT request to update that specific cylinder row in the items table.

3. Safety Checklist Module

Before filling operations begin, a supervisor audits the station. This is a mandatory compliance form.

"The Safety Checklist is a compliance module. Before any filling operation, a supervisor verifies 4 mandatory safety checks — equipment condition, safety valves, fire safety, and PPE compliance. If all checks pass, the overall status is 'Passed'. Each submission creates a new distinct record in the database."

How It Works:

  • Checklist Items: 4 fixed checks (Equipment, Valves, Fire Safety, PPE).
  • Logic: If all checks are "OK", the status is "Passed". If any check is "Not OK", it "Failed".
  • Database Action: Sends a POST request. Unlike QC which updates an item, this inserts a brand new row into the safety_checklists table.

4. Move to Filled Inventory Module

The final step. Cylinders that have been tagged are moved from the filling station to the stock yard.

"This is the final step. After cylinders are sealed and tagged, they move to the stock yard. My module shows all 'Tagged' cylinders, and the operator can move them individually or all at once. Each move records the location, date, and creates a unique transaction ID."

How It Works:

  • Filters Data: Shows cylinders with itemStatus = 'Tagged'.
  • Action: Operator specifies "From" and "To" locations. Can click "Move All" or move one by one.
  • Transaction ID: Generates a unique ID (e.g., TXN-123456) to group all items moved in that specific session.
  • Database Action: Updates item_status = 'In Inventory' and records the location.

5. MySQL & Python Integration Deep Dive

This is the 5-layer journey of how data gets from the browser to the database (using a Quality Check submission as an example):

  1. React Page: Inspector clicks "Submit", firing a function with camelCase data (e.g., gasPurity: 99.5).
  2. Context (Middleman): Converts JavaScript camelCase into Python/SQL snake_case (e.g., gas_purity: 99.5) and sends it as JSON via an HTTP API request.
  3. FastAPI Route: Routes in items.py receive the JSON. A Pydantic Model strictly validates that the data shapes and types are correct before proceeding.
  4. Database Python File: database.py uses mysql-connector-python to open a connection to the local database.
  5. MySQL Execution: The backend dynamically builds an UPDATE or INSERT query and commits the transaction to the sogfusion_db.
Backend Fix Mention: If someone asks about CORS, mention: "We configured CORS middleware in main.py to allow our React app running on port 5173 to safely communicate with our Python API on port 8000."

6. Key Technical Terms Cheat Sheet

Term Simple Meaning
FastAPI The Python framework we used to build our backend server. It's fast and easy to write.
Endpoint / Route A specific Web URL the backend listens to (e.g., /api/batches).
Pydantic Model A Python tool we use that checks if data is correct (e.g., ensures purity is a number, not text).
HTTP Methods GET (fetch data), POST (create new data), PUT (update existing data).
Context (React) A shared data layer in React so different pages can access the same batch information.
JSON The lightweight text format we use to transport data between React and Python.
